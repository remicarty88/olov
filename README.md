# Telegram Бот для учета долгов поставщикам

Бот позволяет вести учет поставщиков, долгов, оплат и просматривать историю платежей. Данные хранятся в Firebase Realtime Database и изолированы для каждого пользователя по его Telegram ID.

## 🚀 Функционал
- 📦 **Поставщики**: Просмотр списка созданных поставщиков.
- ➕ **Добавить поставщика**: Создание нового поставщика.
- 💰 **Добавить долг**: Привязка долга к поставщику с указанием суммы и даты.
- 📊 **Долги**: Список всех долгов (активные и закрытые) с расчетом остатка.
- 💸 **Оплатить**: Внесение оплаты по конкретному долгу.
- 🧾 **История**: Просмотр всех совершенных платежей.

## 🛠 Установка и запуск

1. **Клонируйте репозиторий** (или скачайте файлы).

2. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройте Telegram Бота**:
   - Создайте бота через [@BotFather](https://t.me/BotFather).
   - Скопируйте токен и вставьте его в `bot.py` в переменную `API_TOKEN`.

4. **Настройте Firebase**:
   - Создайте проект на [Firebase Console](https://console.firebase.google.com/).
   - Включите **Realtime Database**.
   - Скопируйте URL базы данных и вставьте в `FIREBASE_DATABASE_URL` в `bot.py`.
   - Перейдите в "Project Settings" -> "Service Accounts" -> "Generate new private key".
   - Скачайте JSON файл, переименуйте его в `firebase-sdk.json` и положите в корневую папку проекта.

5. **Запустите бота**:
   ```bash
   python bot.py
   ```

## 📂 Структура проекта
- `bot.py` — основной файл с логикой бота.
- `requirements.txt` — список необходимых библиотек.
- `firebase-sdk.json` — ключ доступа к Firebase (создается вами).

## 🛡 Безопасность
Данные каждого пользователя хранятся под его уникальным `userId`, что обеспечивает полную изоляцию данных между пользователями.
